Conversão e Coerção de Tipos

Antes de criarmos caminhos e laços, precisamos garantir que estamos trabalhando com os dados certos. Entradas textuais (como as de formulários ou prompts) chegam como String.

Para transformá-las com segurança em tipos numéricos adequados:

  • Number.parseInt(valor, base): Extrai e converte estritamente para um número inteiro.
  • Number.parseFloat(valor): Preserva as casas decimais (números de ponto flutuante).
  • Number(valor): Uma conversão direta e global da linguagem.

O que são Instruções?

As expressões são frases, as instruções são sentenças ou comandos.

Uma expressão produz um valor, enquanto uma instrução é executada para fazer algo acontecer. Por padrão, o interpretador executa o código sequencialmente. As estruturas que veremos hoje quebram esse fluxo linear.

// Uma instrução de expressão (efeito colateral)
let x = 10;

// Uma instrução condicional (altera o fluxo)
if (x > 5) {
    console.log("Fluxo alterado!");
}

A Condicional Simples: if / else

A instrução if é a forma fundamental de controle. Ela avalia uma expressão booleana. Se for verdadeira, executa o bloco. Caso contrário, pula ou entra no bloco else.

if (idade >= 18) {
    console.log("Maior de idade");
} else {
    console.log("Menor de idade");
}

Estruturas Aninhadas: if / else if

Quando existem múltiplos caminhos mutuamente exclusivos, encadeamos cláusulas else if. O JavaScript avaliará cada uma em ordem até encontrar a primeira verdadeira.

let nota = 8.5;

if (nota >= 9.0) {
    console.log("Conceito A");
} else if (nota >= 7.5) {
    console.log("Conceito B"); // Executa este e ignora o resto
} else {
    console.log("Conceito C");
}

Escolha Múltipla: switch

O switch avalia uma expressão e busca uma correspondência exata com uma das cláusulas case. É ideal para simplificar estruturas com muitos else if baseados em valores fixos.

É crucial usar a palavra-chave break para evitar o efeito de transbordamento (fall-through), onde o código continua executando os casos seguintes. A cláusula default é executada se nenhum caso bater.

switch (cor) {
    case "azul":
        console.log("Cor do céu");
        break;
    case "verde":
        console.log("Cor da floresta");
        break;
    default:
        console.log("Cor desconhecida");
}

Laços: while

O laço while é uma estrutura de repetição baseada em pré-teste. A condição é avaliada antes da execução do bloco de código.

Se a condição for inicialmente falsa, o corpo do laço nunca será executado.

let contador = 0;
while (contador < 3) {
    console.log(contador);
    contador++;
}

Laços: do / while

Ao contrário do while clássico, o laço do / while faz o teste da condição no final da iteração. Isso garante que o bloco de instruções seja executado **pelo menos uma vez**, independentemente da condição ser verdadeira ou falsa.

let i = 10;
do {
    console.log("Executa ao menos uma vez!");
    i++;
} while (i < 5);

Laços: for

O laço for centraliza o controle da repetição em três expressões separadas por ponto e vírgula: inicialização, teste de condição e incremento/atualização.

for (let i = 0; i < 5; i++) {
    console.log("Iteração número: " + i);
}

Controle de Laços: break & continue

Podemos alterar drasticamente o fluxo interno de qualquer laço usando instruções de salto:

  • break: Interrompe imediatamente a execução do laço mais interno, saltando para a instrução logo após o loop.
  • continue: Interrompe a iteração atual e pula diretamente para o teste da próxima condição (ou expressão de incremento no laço for).
for (let i = 1; i <= 5; i++) {
    if (i === 3) continue; // Pula o 3
    if (i === 5) break;    // Interrompe o laço ao chegar no 5
    console.log(i); // Imprime: 1, 2, 4
}

Exercícios Práticos (1/2)

Exercício 1 (if / else): Verificador de Login

Crie uma variável contendo uma string recebida. Verifique se o texto é exatamente igual a "admin". Se for, exiba no console ou via alert() "Acesso concedido". Caso contrário, exiba "Acesso negado". Lembre-se de converter entradas se for testar números.

Exercício 2 (if / else if / else): Classificador de Velocidade

Escreva um script que receba um número representativo de uma velocidade. Se a velocidade for menor ou igual a 40, imprima "Velocidade Baixa". Se for maior que 40 e menor ou igual a 80, imprima "Velocidade Ideal". Se for maior que 80, exiba "Velocidade Excessiva".

Exercício 3 (Loops): Tabuada Automatizada

Escolha uma estrutura de repetição (for, while ou do while) e gere no console a tabuada completa do número 7, multiplicando de 1 até 10.

Exercícios Práticos (2/2)

Exercício 4 (break e continue): Filtrando Números

Faça um laço de 1 a 10. Use a instrução continue para pular a impressão do número 5. Além disso, use a instrução break para encerrar completamente o laço se o contador atingir o número 9.

Exercício 5 (Switch Desafiador - Autômatos): O Robô Explorador

Desafio: Imagine um pequeno robô que opera como um autômato de estados simples. Ele possui uma variável de estado chamada estadoAtual (inicialmente "parado"). Crie um switch que leia strings de comandos recebidos: "LIGAR", "MOVER", "BLOQUEAR" e "DESLIGAR".

Desenhe as regras: Se o comando for "LIGAR", o robô passa para o estado "alerta". Se for "MOVER" e o estado for "alerta", ele passa para "andando". Se o comando for "BLOQUEAR", ele vai para "emergencia". Trate cenários inválidos usando a cláusula default para emitir um aviso de comando desconhecido!

Gabarito

Sugestão de implementação para o Autômato do Robô:

let continuar = true;
while (continuar) {
    // let é parecido com "var"
    // na verdade, let é a forma moderna de declarar variáveis em js
    // e indica o escopo* da variável
    let estadoAtual = "alerta"; 
    let comando = prompt(
        "Digite um comando para o robô:\n" +
        "LIGAR | MOVER | BLOQUEAR | DESLIGAR"
    );

    switch (comando) {
        case "LIGAR":
            estadoAtual = "alerta";
            alert("Robô pronto. Estado: " + estadoAtual);
            break;

        case "MOVER":
            if (estadoAtual === "alerta") {
                estadoAtual = "andando";
                alert("Robô em movimento. Estado: " + estadoAtual);
            } else {
                alert("Não posso mover. Robô não está em alerta!");
            }
            break;

        case "BLOQUEAR":
            estadoAtual = "emergencia";
            alert("Alerta máximo! Estado: " + estadoAtual);
            break;

        case "DESLIGAR":
            estadoAtual = "parado";
            alert("Desligando... Estado: " + estadoAtual);
            break;

        default:
            alert("Comando não reconhecido pelo autômato.");
    }

    // Pergunta se quer repetir o programa
    let resposta = prompt("Deseja executar novamente? (SIM / NAO)");

    if (resposta !== "SIM") {
        continuar = false;
        alert("Programa encerrado.");
    }
}

Dicas de Ouro

  • Cuidado com Loops Infinitos: Certifique-se sempre de que a expressão de teste em um laço while ou for eventualmente se tornará falsa.
  • O Papel do Break: Se esquecer o break dentro do switch, o JavaScript continuará executando todos os blocos seguintes sem testar as condições!
  • Coerção explícita: Sempre prefira converter explicitamente dados textuais com Number.parseInt() antes de efetuar comparações numéricas rígidas.
