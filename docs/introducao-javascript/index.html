O Guia do Comportamento Web

Role para baixo para começar

A Trindade do Desenvolvimento

  • HTML5: A Estrutura (Os ossos do site).
  • CSS3: O Estilo (A pele, roupas e cores).
  • JavaScript: O Comportamento (O cérebro e os movimentos).

Sem JS, a web é estática. Com JS, ela ganha vida de verdade.

Papéis Claros

O que já vimos:

Você já sabe criar botões com HTML e estilizá-los mudando cores e fontes com CSS.

O JavaScript agora vai ditar a ação: o que esse botão faz quando é clicado.

A Grande Diferença:

  • HTML/CSS: Linguagens Declarativas (Definem o que a página é).
  • JavaScript: Linguagem Lógica (Define o que acontece e quando acontece).

De onde vem o Código?

Quando digitamos um endereço web como www.google.com, um processo invisível acontece:

  • Seu navegador envia um pedido chamado Requisição.
  • Um servidor remoto localiza onde os arquivos estão guardados.
  • O servidor gera e envia uma Cópia exata desses arquivos para a sua máquina.

O Catálogo: DNS

O DNS (Domain Name System) funciona como a lista de contatos do celular:

  • Computadores usam números complexos chamados IPs (ex: 142.250.191.100).
  • O DNS traduz o nome amigável "google.com" para o IP real do servidor.
  • Com o endereço correto, o navegador recebe o HTML e exibe na tela.

Lado do Cliente (Frontend)

Embora o JavaScript seja incrivelmente potente e possa rodar no Servidor (usando Node.js), nosso foco total agora será no Client-side.

Fato Crítico para Entender:

O código JavaScript que vamos escrever é interpretado e rodado diretamente dentro do navegador do seu próprio usuário, e não lá no servidor distante.

O servidor apenas despacha o arquivo de texto bruto; o processamento real acontece no computador de quem acessa.

Ambiente de Trabalho

  • VS Code: Nosso ambiente de desenvolvimento inteligente que vocês já usam.
  • Live Server: A extensão que simula esse servidor local atualizando a tela a cada salvamento.
  • Console do DevTools: O painel secreto do programador JavaScript (basta apertar F12 no navegador).

A Linguagem do Navegador

O JavaScript foi estruturado especificamente para dar dinamismo às páginas. Suas principais marcas são:

  • Interpretada: O navegador lê e executa os comandos em tempo real, linha após linha.
  • Dinâmica: Consegue reescrever elementos HTML e regras CSS enquanto a página já está aberta.
  • Baseada em Eventos: Fica atenta e "escutando" as interações do usuário, como cliques e movimentos do mouse.

Comando: alert()

O comando alert() cria um aviso direto na tela bloqueando as interações até que o usuário clique em 'OK'.

Como escrever:

alert("Olá, mundo!");

Perfeito para validar se o arquivo JS está conectado ou exibir alertas urgentes.

Armazenando Dados

Variáveis funcionam como caixas organizadoras onde guardamos informações identificadas por etiquetas.

  • let: Usado para informações que podem mudar ao longo do tempo.
  • const: Usado para valores que devem permanecer fixos (constantes).
let nome = "João";
const idade = 16;

Comando: querySelector

Como o código JavaScript consegue capturar um elemento criado no seu arquivo HTML?

Utilizamos os mesmos seletores que aprendemos no CSS!

const titulo = document.querySelector("h1");

A partir dessa linha, a variável titulo controla diretamente a tag H1 do site.

Mudando o Estilo

Conseguimos alterar as propriedades visuais manipulando o atributo style de qualquer elemento selecionado.

// Seleciona o plano de fundo da página completa
const corpo = document.querySelector("body");

// Altera instantaneamente a cor de fundo para preto
corpo.style.backgroundColor = "black";

Atenção: Propriedades CSS com hífen como background-color passam a ser escritas em formato CamelCase no JavaScript: backgroundColor.

Respondendo a Ações

Para disparar uma alteração quando o usuário interagir, monitoramos o evento de clique (onclick):

// Captura o botão do HTML
const botao = document.querySelector("button");

// Define a função que será executada ao clicar
botao.onclick = function() {
  alert("O botão foi acionado!");
};

Basics vs Frameworks

Estruturas e bibliotecas famosas do mercado como React, Vue e Angular são ferramentas robustas e muito utilizadas.

Mas lembre-se: todas elas são construídas usando puramente o JavaScript básico!

Sem dominar a lógica pura, variáveis e os conceitos de manipulação básicos, você vira apenas um replicador de códigos prontos sem entender o que faz.

Gabarito Sugestão

Aqui está o código JavaScript de resposta que resolve os desafios de manipulação de DOM (Exercícios 3, 4 e 5):

// --- RESOLUÇÃO DOS EXERCÍCIOS 3 E 4 ---
const btnCor = document.querySelector("#btnCor");
const titulo = document.querySelector("h1");

btnCor.onclick = function() {
  document.body.style.backgroundColor = "red";
  titulo.innerText = "JavaScript é Incrível!";
};

// --- RESOLUÇÃO DO EXERCÍCIO 5 (CONTADOR) ---
let contador = 0;
const display = document.querySelector("#count");
const btnContador = document.querySelector("#btnCount");

btnContador.onclick = function() {
  contador++;
  display.innerText = contador;
};

Dicas de Ouro

  • Sempre feche suas strings: Textos brutos sempre precisam estar delimitados por aspas duplas " " ou aspas simples ' '.
  • Letras maiúsculas mudam tudo: O JavaScript diferencia maiúsculas de minúsculas. Escrever queryselector com o "s" minúsculo vai quebrar seu código. O correto é querySelector.
  • Inspeção interna: Ficou na dúvida se o valor mudou? Use o comando console.log(variavel) para verificar os dados direto na aba Console do DevTools.
