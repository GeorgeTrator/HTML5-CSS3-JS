O que é um Formulário?

Um formulário é a forma padrão de coletar dados do usuário: login, cadastro, busca, comentário — tudo isso são formulários.

Ele vive dentro da tag <form> e é composto por campos (<input>) e um botão de envio.

  • action: para onde os dados vão.
  • method: como os dados são enviados.
<form action="/cadastro" method="post">
    <label>Nome:</label>
    <input type="text" name="usuario">

    <button type="submit">Enviar</button>
</form>

GET vs POST

O atributo method define o "jeito" de enviar os dados. Existem dois principais:

GET

  • Dados vão na URL (após o ?).
  • Visíveis, ficam no histórico e podem ser favoritados.
  • Ideal para buscas e filtros.
  • Não use para senhas!

POST

  • Dados vão no corpo da requisição.
  • Não aparecem na URL.
  • Ideal para cadastros, login e envio de dados sensíveis.
  • Sem limite prático de tamanho.

Resumo: GET = "pedir para ver". POST = "enviar para guardar".

Tipos de Campo (input)

O atributo type muda o comportamento do campo. Os principais:

<!-- Texto simples (nome) -->
<input type="text" name="nome"
       placeholder="Seu nome"
       required maxlength="30">

<!-- Senha (esconde os caracteres) -->
<input type="password" name="senha"
       required minlength="6">

<!-- Data -->
<input type="date" name="nascimento">

<!-- Número, com limites -->
<input type="number" name="idade"
       min="0" max="120">

Atributos importantes

  • required: obriga o preenchimento.
  • placeholder: texto de dica dentro do campo.
  • maxlength / minlength: limite de caracteres (texto).
  • min / max: valor mínimo e máximo (número e data).
  • name: o "nome" com que o dado chega no servidor. Sem ele o campo não é enviado!

Formulários dependem do navegador

Cada tipo de input é renderizado pelo navegador, e não pelo seu código. Isso significa que o mesmo campo pode aparecer diferente (ou nem funcionar direito) dependendo de onde o usuário abre a página.

O caso mais famoso é o Safari de Desktop (macOS), que historicamente teve problemas para exibir campos como type="date" — muitas vezes mostrando um campo de texto comum em vez do seletor de calendário.

Lição prática: sempre teste seu formulário em mais de um navegador (Chrome, Firefox, Safari, Edge) antes de considerá-lo pronto.
